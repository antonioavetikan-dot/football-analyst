# Pro Football Analyst AI ⚽

Сайт с анализом футбольных матчей дня и низкорисковыми ставочными инсайтами.
Прогнозы **рассчитываются движком** из формы команд, личных встреч, xG-модели,
угловых и фактора домашнего поля — не берутся случайно. Уровень риска и уверенность
показаны открыто по каждому рынку.

> Аналитический инструмент, **не** гарантия результата и не финансовый совет. 18+.

---

## Что внутри

```
index.html                      — весь сайт (HTML+CSS+JS, без фреймворков)
data/matches.json               — данные матчей (их перезаписывает скрипт раз в день)
scripts/fetch-matches.mjs       — тянет матчи из API и считает прогнозы
.github/workflows/update-matches.yml — авто-обновление раз в день
```

Если `data/matches.json` пустой или недоступен — сайт автоматически показывает
встроенный демо-набор, так что он работает сразу, ещё до подключения API.

---

## Запуск локально (без API)

Просто откройте `index.html` в браузере — увидите демо.
Если хотите проверить загрузку из `data/matches.json`, поднимите локальный сервер:

```bash
cd site
python3 -m http.server 8080
# откройте http://localhost:8080
```

(через `file://` браузер блокирует `fetch` к json — поэтому нужен мини-сервер).

## Бесплатный хостинг + ежедневное авто-обновление (GitHub Pages)

Выбран GitHub Pages, потому что **сайт и планировщик обновления живут в одном месте** —
ничего больше платить и настраивать не нужно.

### Шаг 1. Бесплатный API-ключ
1. Зарегистрируйтесь на **https://www.football-data.org/client/register** (бесплатно, без карты).
2. Скопируйте ваш `API token`.
   Бесплатный тариф: 12 топ-турниров (АПЛ, Ла Лига, Бундеслига, Серия А, Лига 1,
   Лига чемпионов, Эредивизи, Примейра, Чемпионшип, Бразилия и др.), 10 запросов/мин.
   *России/СНГ в бесплатном тарифе нет — про них см. раздел ниже.*

### Шаг 2. Репозиторий
1. Создайте новый репозиторий на GitHub (например `football-analyst`).
2. Загрузите в него содержимое папки `site/` (файлы должны лежать в корне репозитория).

### Шаг 3. Добавить ключ в секреты (он не попадёт в код)
`Settings → Secrets and variables → Actions → New repository secret`
- Name: `FOOTBALL_API_KEY`
- Secret: ваш токен из шага 1.

### Шаг 4. Включить GitHub Pages
`Settings → Pages → Build and deployment → Source: Deploy from a branch`
- Branch: `main`, папка `/ (root)` → Save.
- Через минуту сайт будет по адресу `https://ВАШ_ЛОГИН.github.io/football-analyst/`.

### Шаг 5. Запустить обновление
`Actions → «Обновление матчей дня» → Run workflow` (первый раз вручную).
Дальше экшен сам запускается **каждый день** (по умолчанию 06:00 UTC) и коммитит
свежий `data/matches.json`. Время правится в `.github/workflows/update-matches.yml`
(строка `cron`).

Готово — матчи обновляются автоматически, ключ скрыт, всё бесплатно.

## Альтернатива: Netlify / Cloudflare Pages

Сайт статический, поэтому зальётся и туда (перетащить папку или подключить репозиторий).
Но **планировщик** (ежедневный запуск скрипта) там настраивается отдельно
(Netlify Scheduled Functions / Cloudflare Cron Triggers). На GitHub Pages это
уже включено через Actions, поэтому он проще для вашей задачи.

## Россия/СНГ и «все лиги мира»

`football-data.org` в бесплатном тарифе их не покрывает. Для них подойдёт
**API-Football** (RapidAPI), бесплатно 100 запросов/день, есть все 1200+ лиг.
Чтобы переключиться: впишите ключ в секрет, поменяйте в `fetch-matches.mjs`
адрес `BASE`, заголовки авторизации и разбор ответа под формат API-Football —
структура объекта матча и движок (`analyze`) остаются прежними.

## Подключение реальной статистики (точнее прогнозы)

Сейчас для real-time данных голы/угловые оцениваются грубо из формы (заглушка),
т.к. бесплатный тариф не отдаёт детальную статистику. Чтобы прогнозы стали точнее,
заполните в объекте матча реальные средние:
`home.gf, home.ga, home.corners`, аналогично `away`, и `h2h`.
Движок (`Engine.analyze`) сразу пересчитает исход, тотал, обе забьют, угловые и риск.

---

## Как работает движок (кратко, без «магии»)

1. **Ожидаемые голы**: атака команды + слабость обороны соперника, поправка на дом/гостей.
2. **Исход**: перевес = разница xG + разница формы + перевес по личным встречам.
3. **Тотал / обе забьют / угловые**: из суммарных ожидаемых голов и средних угловых.
4. **Риск**: `Низкий` если самый уверенный рынок ≥70%, `Средний` ≥60%, иначе `Высокий`.
5. **Стратегия**: только низкорисковые ставки, лестница банка +5% за шаг, 1–2 ставки в день,
   шаг вниз при проигрыше, без догона.

Никаких «гарантированных выигрышей» — только вероятность и управление банком.
